**Test Plan**
**Website** : Saucedemo - https://www.saucedemo.com/ 

  **1. Purpose**
The purpose of the Swag Labs website is to provide an online platform where users can view, select, and purchase available products. 

 **2. Project scope:**
 - *Login:* Verification of valid and invalid credentials.  
 - *Main Page*: Verification of product images, prices, and product information.
 - *Filtering:* Verification of filtering products by name and price.   
 - *Burger Menu:* Verification of menu items including About, Logout, and 
   Reset App State. 
- *Cart Functionality*:
  - Adding products to the cart
  - Removing products from the cart
  - Proceeding to checkout
  - Returning to the main page
    
 - *Footer / Social Media Links:* Verification of social media links (LinkedIn, Facebook, Twitter).

 3. **Out of Scope:**
 - Payment processing
 - Backend and database testing
 - Security testing

4.	**Types of Testing**
- Functional testing
- Non-Functional Testing
  - Performance Testing (Load Testing)
- UI/UX Testing
- Cross Browser Testing

5. **Entry Criteria**
- The website is accessible.
- Test cases are prepared in Qase.io.
- Test credentials are available.

6. **Exit Criteria** 
- All planned test cases are executed.
- Critical defects are fixed or reported.
- Test results are documented.
- Testing is completed within the defined timeline.

7.	**Test Environment**
• Browser:
  - Google Chrome (latest version)
  - Microsoft Edge (latest version)
  - Opera (latest version)
• Operating System: Windows 10

10.	**Risks**
- Some cart and filtering functions may not be fully verified due to time constraints.
- Layout or buttons may display incorrectly on different browsers.
- Unexpected browser or OS updates may cause false test failures.
