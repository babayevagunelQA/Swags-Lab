# Swag Labs – Test Scenarios

## TS-01: Successful Login

Registered user logs in with valid credentials and is redirected to the Products page.

## TS-02: Invalid Login Attempt

Error message is displayed when an invalid username or password is entered.

## TS-03: Locked User Login

Locked user cannot log in and receives the appropriate error message.

## TS-04: View Products

All products are displayed correctly with name, image, description, and price on the Products page.

## TS-05: Add Product to Cart

User adds a product to the cart and the cart badge count increases accordingly.

## TS-06: Remove Product from Cart

User removes a product from the cart and the cart badge count decreases.

## TS-07: Product Sorting

Products can be sorted by name (A–Z, Z–A) and price (low to high, high to low).

## TS-08: Burger Menu Navigation

All burger menu options (All Items, About, Logout, Reset App State) function correctly.

## TS-09: Cart Functionality

Selected products are displayed correctly on the Cart page with accurate details.

## TS-10: Checkout Validation

Error messages are displayed when required checkout fields are left empty.

## TS-11: Successful Checkout

User completes the checkout process successfully and sees the order confirmation message.

## TS-12: Logout Security

After logout, the user is redirected to the Login page and cannot access protected pages using the browser back button.

## TS-13: Refresh Behavior

Products added to the cart remain in the cart after refreshing the page.

## TS-14: Direct URL Access

User cannot access protected pages (e.g., inventory page) directly via URL without logging in.

## TS-15: Multiple Tabs Behavior

User session remains active and consistent when the website is opened in multiple browser tabs.

## TS-16: Add Same Product Twice

User cannot add the same product multiple times using the "Add to Cart" button.

## TS-17: Browser Back Navigation Flow

User is navigated to the previous page correctly when using the browser back button.

## TS-18: Checkout With Invalid Data Format

System displays validation errors when invalid data formats are entered in checkout fields.

## TS-19: UI Validation of Error Messages

Error messages are displayed clearly and correctly in the user interface when an action fails.
